How {{ APP.name }} works?


1. WBS definition

The WBS can be imported from an interface or updated by authorized users. When the list of activities has been loaded, is responsibility of Project Control to determine if each activity can receive hours of direct or indirect types. This process is done only once because is very rare that an activity that is supposed to receive direct hours can change later on to receive only indirect hours. So the definition of the type of hours is done at the beginning of the project only but the tool also allows to introduce changes later on if needed

2. Manpower Resources

The manpower can also be imported or added manually by authorized users. The source of data can be an HR or payroll database that will be able to produce the list of workers currently active. This list is updated manually or thorough the same interface in order to include new hired staff or to remove those that have been terminated. After the updating of the manpower data the table of job titles shall be updated by identifying on each job title which type of hours the position will be able to book worked time

3. Equipment Resources

The equipment can as well be imported or updated by authorized users. The source is usually the assets table that exists in the accounting or ERP system the company is using to support their business. The import of equipment from the interface is usually done only once at the beginning of the project. There is need to import the interface again only when there is new equipment being assigned to the project.

4. Teams Definition

A team is a working unit. It is composed by staff and, sometimes, equipment. A team includes those workers and equipment that are usually assigned to activities that they must be together to perform. In a workshop there could be one or more teams involved in several activities. Per instance, a welding team. It will include a foreman or supervisor, some welders, grinders, helpers, etc. The staff working in HR and responsible for the payroll are also part of a team. The protocol staff in an organization are a team. The concept of team helps to normalize the input of data for timekeeping purposes. A team must have a timekeeper assigned and at least one approver for its timesheet. A team will be allowed to charge hours to specific project activities that are defined usually by the planner of the project. Inside this tool a team is a working unit that charges hours to predefined activities only.

5. Timesheet Entry and Approval

The timekeeper assigned to input man and equipment hours for the team will do so in a daily basis. Once the timesheet has been completed the timekeeper will send it for review. This is done electronically and the user assigned to review that team will be notified immediately. If there are more than one approver/reviewer assigned to a team all of them will be notified at once so they can approve online when they consider it convenient. As the team is allowed to charge hours to specific activities assigned by the project, is not possible for the timekeeper to enter hours in activities where the team is not supposed to work in. Once the timesheets are approved they are available for further consultation whenever is needed. The timesheet will track all the users that intervened from the creation until the final approval so it is easy to trace user actions performed in a particular timesheet

6. Payroll Interfaces

At the end of the payroll period authorized users linked to payroll activities will be able to download from this tool the total worked hours for the staff. The download tool will generate a file (csv, flat file, etc.) that can be imported into the payroll software for salary calculation (payroll software permitting). So you are going to pay salaries based on actual approved worked hours which helps preventing the existence of ghost workers and unauthorized worked hours.

After processing the payroll calculation it will also be possible (again, payroll software permitting) to extract the actual labor cost per staff and then upload it into this tool in order to distribute actual cost by activity. That information can then be downloaded from this tool and then uploaded into the ERP system the company is using. Thus to reduce error margin in the input of data related to Project and Job Costing.

7. Scheduling Software Interface

As this tool helps collecting the actual worked time spent into the different activities of the project, it also allows exporting the summary of working time per activity that can later on be uploaded into your project scheduling software (i.e. : Oracle Primavera P6 ®) so you will have your progress updated without having to worry about the accuracy of the info as all loaded hours were duly approved
